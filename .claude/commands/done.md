## タスク

引数: `$ARGUMENTS`

以下の手順で作業完了処理を行ってください。

1. 引数からS番号を特定する（例: `S003` または `003` → `S003`）
2. `steering/` から該当のステアリングファイル（`S###-*.md`）を特定して読み込む

### ステップA: サブエージェントによる並列品質チェック

ステータス更新の前に、以下の**2つのサブエージェントを並列で**起動してください。

**サブエージェントA — Playwright テスト（Bash エージェント）**
- 実行コマンド: `npx playwright test`
- テスト結果（パス数・失敗数・エラー内容）を収集する

**サブエージェントB — スペック適合チェック（general-purpose エージェント）**
- ステアリングファイルの「完了条件」と「影響ファイル」を読み込む
- 影響ファイルの実装内容と完了条件を照合し、未達成項目を列挙する

### ステップB: 判定

両エージェントの結果を受け取り、以下で判定する:

- **全パス かつ 未達成完了条件なし** → ステップCへ進む
- **問題あり** → 完了宣言を保留し、問題点を報告してユーザーに修正を依頼する

### ステップC: ファイル更新（品質チェック通過後のみ）

3. ステアリングファイルを以下のとおり編集する:
   - `**ステータス**: 進行中` → `**ステータス**: 完了` に変更
   - `**完了日**: —` → 今日の日付に変更
   - スコープの `- [ ]` チェックボックスをすべて `- [x]` に変更
   - 完了条件の `- [ ]` チェックボックスをすべて `- [x]` に変更
4. `docs/design.md` を読み込み、該当機能のステータスや記述があれば「完了」に更新する
   - 明示的な対応箇所がない場合は「design.md に追記すべき内容があれば教えてください」と案内する
5. 完了後に以下を出力する:
