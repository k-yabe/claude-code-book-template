## コンテキスト

ステアリングファイル一覧:
```
!ls steering/S*.md 2>/dev/null | sort
```

## タスク

以下の手順でレビューを実施してください。

### ステップ1: 進行中ファイルの特定

`steering/` 内の全 `S*.md` を読み込み、**ステータスが「進行中」かつ最新番号**のファイルを1件特定して読み込む。
存在しない場合は「進行中のステアリングファイルはありません。`/new-spec <タイトル>` で新しい作業を開始してください。」と出力して終了。

---

### ステップ2: 3つのサブエージェントを並列起動

ステアリングファイルの内容を読み込んだ上で、以下を**同時に**起動する。

---

**サブエージェントA — スペック品質チェック（general-purpose）**

以下の観点でステアリングファイルを査読し、問題点を列挙する。実務レベルで厳しく評価すること。

1. **完全性**: 目的・スコープ・完了条件・影響ファイルに空欄・「TBD」・曖昧な記述がないか
2. **完了条件のテスト可能性**: 各完了条件が「Yes/No で判定できる具体的な基準」になっているか。「〜できる」だけでは不十分—何をもって完了とするかが明確か
3. **スコープの境界**: やること／やらないことが明確に分離されているか。実装者が迷わないか
4. **抜け漏れリスク**: エラーハンドリング・認証・セキュリティ・レスポンシブ対応・ローディング状態など、よくある実装漏れがスコープに含まれているか
5. **前提条件の明示**: 外部依存（APIキー・ライブラリ・テンプレートファイル等）が明示されているか

問題なければ「✅ スペック品質: 問題なし」と出力。問題があれば箇条書きで指摘する。

---

**サブエージェントB — 技術実現可能性チェック（general-purpose）**

スペックに記載された技術要素を調査し、実現可能性を評価する。

1. **ライブラリ確認**: 使用予定のライブラリ（JSZip, PptxGenJS, Chart.js 等）がCDN配信されているか、ブラウザのみで動作するか、バージョン制約はないか
2. **API制約**: Claude API の利用方法（モデル名・エンドポイント・レスポンス形式）に誤りはないか
3. **ブラウザ制約**: 使用するWeb API（File API, Canvas API, Blob, ArrayBuffer等）にブラウザ互換性の問題はないか
4. **技術的難易度**: 「できる」と書かれていても実際には困難・不安定な要素はないか（例: SVG生成の品質ばらつき、PPTXテンプレートのXML構造への依存等）
5. **パフォーマンス**: 重い処理（画像変換・大量XML操作等）でUIがブロックされるリスクはないか

問題なければ「✅ 技術実現可能性: 問題なし」と出力。問題があれば代替案とともに指摘する。

---

**サブエージェントC — コードベース整合チェック（Explore）**

`apps/` 配下の既存アプリ（特に直近のもの）を読み込み、以下を確認する。

1. **UI一貫性**: ナビバー・ボタン・カラー・フォント・フッターのパターンが既存アプリと一致しているか。スペックに記載のない必須要素がないか
2. **ブランドガイドラインNG**: border-radius / 色付き枠線 / 色付き図形+枠線 のNG要素がスペックに含まれていないか
3. **認証チェック**: `localStorage.getItem('auth') !== '1'` によるリダイレクト処理がスコープに含まれているか
4. **影響ファイルの漏れ**: スペックの「影響ファイル」に記載がないが、実際には変更が必要なファイル（index.html の WHAT'S NEW・カード追加など）がないか
5. **命名・構造**: `apps/<app-name>/index.html` の命名規則に従っているか

対象: `apps/banner-resizer/index.html`, `apps/sns-post-generator/index.html` を中心に確認。

問題なければ「✅ コードベース整合: 問題なし」と出力。問題があれば具体的なファイル名・行番号とともに指摘する。

---

### 判定

- **指摘なし** → 「✅ 全チェック通過。承認しますか？」
- **軽微な指摘のみ** → 「⚠️ 軽微な指摘があります。修正して承認しますか？それとも指摘を無視して進めますか？」
- **重大な指摘あり** → 「🚨 重大な問題があります。実装前にスペックの修正を推奨します。」
