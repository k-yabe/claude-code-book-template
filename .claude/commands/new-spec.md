## コンテキスト

テンプレート:
```
!cat steering/_template.md
```

既存のステアリングファイル一覧:
```
!ls steering/S*.md 2>/dev/null | sort
```

## タスク

引数: `$ARGUMENTS`

以下の手順で新しいステアリングファイルを生成してください。

1. 既存の `steering/S*.md` ファイルの番号一覧から最大番号を特定し、次の番号（3桁ゼロ埋め）を決定する
   - 例: S001, S002 が存在する場合 → 次は S003
2. ファイル名を `steering/S###-<引数をケバブケースに変換>.md` に決定する
   - 引数が日本語の場合はそのまま使う（例: `S003-新機能の追加.md`）
3. `_template.md` の内容をベースに以下を埋めて新規ファイルを作成する:
   - `[S000]` → 決定した番号（例: `[S003]`）
   - タイトル部分 → 引数の内容
   - `YYYY-MM-DD` → 今日の日付（2026-02-25）
   - ステータスは `進行中` に設定する
4. 作成したファイルのパスと内容を出力して確認を求める
5. ステアリングファイル作成後、**Explore サブエージェントを自動起動**して以下を調査・提案させる:
   - `apps/` 配下の既存アプリ（HTML）のUIパターンを分析（色・コンポーネント・レイアウト）
   - 新しいアプリタイトル「$ARGUMENTS」に対して、追加すべき機能・UIの一貫性チェック・抜け漏れを提案
   - サブエージェントへの指示: 「`apps/` 配下の既存HTMLアプリを読み込み、共通UIパターン（ナビバー・カード・ボタン・カラーパレット）を把握した上で、新アプリ『$ARGUMENTS』の実装前チェックリストを出力してください。①UI一貫性のために守るべき点、②マーケターが実際に使う上で追加すると良い機能、③よくある実装漏れ、の3点で箇条書きで回答してください。」

作成後: サブエージェントの提案を踏まえてスペックを記述し、`/review` でレビューを依頼するよう案内する。
