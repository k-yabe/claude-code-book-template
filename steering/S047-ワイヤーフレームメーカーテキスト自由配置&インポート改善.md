# [S047] Wireframe Maker テキスト自由配置 & インポート改善

## 目的

Wireframe Makerの致命的なUX課題2点を修正する。①テキストを好きな場所に自由配置できないため、ユーザーがレイアウトをカスタマイズできない。②URLインポート（WF変換）が既存セクションを全消しするため、作業中のワイヤーフレームに追加取り込みができない。

---

## スコープ

### やること
- [x] テキスト要素の自由配置機能（キャンバス上の任意の位置にテキストを追加・移動）
- [x] URLインポート時に既存セクションを保持するオプション（追加/置換の選択）

### やらないこと（スコープ外）
- 画像要素の自由配置（将来の作業単位で扱う）
- セクション以外のフリーキャンバスモード全般

---

## 完了条件

- [x] キャンバス上の任意の位置にテキスト要素を追加できる
- [x] 追加したテキストをドラッグで移動できる
- [x] テキストのフォントサイズ・色・太さを変更できる
- [x] URLインポート時に「既存に追加」か「全て置換」かを選択できる
- [x] 既存セクションがある状態でURLインポートしても消えない（追加モード時）

## 作業ログ

### 2026-04-04
- **テキスト自由配置**: `freeTexts`配列をページデータに追加。T+ツールバーボタンでテキスト配置モード→キャンバスクリックで任意位置にテキスト作成→ドラッグ移動・ダブルクリック編集・クリックでツールバー（サイズ/太さ/色/削除）
- **インポート改善**: `handleUrlImport()`と`generateFromImport()`に「追加/置換」選択モーダルを追加。既存セクションがある場合のみ表示
- **バグ修正**: Undo/Redo状態にfreeTextsを含めるよう修正。マルチページ切り替え・削除時のfreeTexts同期を追加。プロトタイプモード開始時のfreeTexts同期を追加
- **設計判断**: freeTextsはセクションとは独立したSVGレイヤーとして描画。セクション上に重ねて配置可能
