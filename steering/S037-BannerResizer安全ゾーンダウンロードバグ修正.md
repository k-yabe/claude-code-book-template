# [S037] Banner Resizer: ダウンロード画像に安全ゾーンが含まれるバグ修正

**ステータス**: 進行中 <!-- 未着手 | 進行中 | レビュー中 | 完了 -->
**作成日**: 2026-03-31
**完了日**: —

---

## 目的

Banner Resizerでバナー（MV）サイズをダウンロードすると、プレビュー用の安全ゾーンオーバーレイ（半透明赤・緑破線・ラベル）が画像に焼き込まれた状態で出力されるバグを修正する。

---

## スコープ

### やること
- [ ] ダウンロード時に安全ゾーンオーバーレイを除外してクリーンな画像を出力する
- [ ] プレビュー表示では引き続き安全ゾーンオーバーレイを表示する

### やらないこと（スコープ外）
- 安全ゾーンのサイズ値変更やUI改善
- 新しいサイズの追加

---

## 原因分析

プレビュー用キャンバス（`canvas-{sizeId}`）に `drawSafeZoneOverlay()` で直接オーバーレイを描画しており、ダウンロード時にも同じキャンバスから `canvas.toBlob()` で出力するため、オーバーレイが含まれてしまう。

**該当コード（行895-902）**:
```javascript
if (!needsAI) {
  const canvas = createResized(sourceImage, size.w, size.h);
  canvas.id = 'canvas-' + size.id;
  if (size.safeZone) {
    drawSafeZoneOverlay(canvas, size.safeZone);  // ← ここで焼き込み
  }
  document.getElementById('wrap-' + size.id).appendChild(canvas);
}
```

**修正方針**: ダウンロード時に元画像から安全ゾーンなしでキャンバスを再生成してからBlobに変換する。

- [ ] ZIP一括ダウンロードでも同様に安全ゾーンが含まれないこと
