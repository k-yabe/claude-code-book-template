## 目的

Wireframe Maker の UI/UX を総合的に見直し、マーケターが日常的に使えるプロフェッショナルツールとしての完成度を高める。

---

## スコープ

### やること（11項目）

#### A. チャットパネル改善
- [ ] **A1: ウェルカム画面の刷新** — チャット初期状態を「空のチャット」→「使い方カード形式」に変更。3つのカード（URLコピー / テンプレ選択 / 自由入力）で初見ユーザーを誘導
  - 受け入れ基準: チャット未使用時に3枚のカードが表示され、各カードクリックで対応フローが起動すること
- [ ] **A2: URLインポートのUI改善** — 現在の「コピー生成」縦書きボタンが分かりにくい。インポートゾーンを横一列レイアウトに整理し、ドラッグ＆ドロップエリアとURLバーを見やすく
  - 受け入れ基準: URLバーとファイルドロップが1行に収まり、ボタンが横書きで読めること

#### B. 構成パネル改善
- [ ] **B1: セクションカードの情報密度向上** — CVRバッジの視認性UP、セクション順序番号の追加、ホバー時のアクションボタン表示改善
  - 受け入れ基準: 各カードに連番が表示され、CVRバッジがテキストラベル付きになること
- [ ] **B2: セクション追加UIの改善** — プルダウン→ビジュアルピッカー（アイコン+名前のグリッド）に変更
  - 受け入れ基準: 「+追加」クリックでグリッドポップアップが表示され、アイコン付きでセクション型を選択できること

#### C. ツールバー改善
- [ ] **C1: ツールバーアイコンにツールチップ追加** — 現在アイコンのみで意味が不明なボタンが多い。title属性+CSSツールチップで補足
  - 受け入れ基準: 全ツールバーボタンにホバーで日本語ツールチップが表示されること
- [ ] **C2: デバイス切替のビジュアル改善** — PC/Tab/SP の表示幅を明示（1440px / 768px / 375px）
  - 受け入れ基準: デバイスボタンに幅数値が併記されること

#### D. エクスポートパネル改善
- [ ] **D1: エクスポート画面の刷新** — 各フォーマットにアイコン+説明を追加。HTMLコードエクスポートも追加
  - 受け入れ基準: SVG/PNG/PDF/HTMLの4フォーマットが個別カードで表示され、HTMLエクスポートでコードがコピーできること
- [ ] **D2: プロジェクト管理UIの統合** — 保存/開く/JSON入出力をエクスポートタブ内にも配置
  - 受け入れ基準: エクスポートタブに保存・開く・JSONエクスポート・JSONインポートの4ボタンが配置されていること

#### E. 全体UX改善
- [ ] **E1: キーボードショートカットの拡充+ヘルプ改善** — 既存ショートカット表に不足分を追加、?キーで呼び出し可能に
  - 受け入れ基準: ?キーでショートカット一覧が表示され、Ctrl+Z/Y/S/E等が一覧に含まれること
- [ ] **E2: Undo/Redo の動作改善** — saveUndoState()の呼び出し漏れを修正。対象: セクション追加・削除・移動・テキスト編集・スタイル変更
  - 受け入れ基準: 上記5操作それぞれでCtrl+Z→元に戻る、Ctrl+Shift+Z→やり直しが動作すること

#### F. UI一貫性修正
- [ ] **F1: ナビバー高さを他アプリと統一** — 48px→58pxに変更し、editor-layoutの高さ計算を調整
  - 受け入れ基準: ナビバーが58pxになり、レイアウト崩れがないこと

### やらないこと（スコープ外）
- バックエンドAPI の変更
- 新しいセクションタイプの追加
- リアルタイムコラボレーション
- Figma直接連携
- ツール自体のモバイルレスポンシブ対応（既存のまま）
