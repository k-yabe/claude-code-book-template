**完了日**: 2026-04-05

---

## 目的

Wireframe Makerをユーザー目線で使用した際に致命的なUX問題が5件見つかったため修正する。

---

## スコープ

### やること
- [x] Bug1: 比較表セクションの `Math.random()` による表示の不安定さを修正
- [x] Bug2: `onSectionsChanged` がチャット中でも強制的に構成タブに切り替える問題を修正
- [x] Bug3: ショートカットパネルの Ctrl+S 説明が「SVGダウンロード」と誤表記（実際はプロジェクト保存）
- [x] Bug4: 自動復元時に `updateCVRDisplay()` が呼ばれずCVRスコアが表示されない
- [x] Bug5: `useMarketTemplate()` が `onSectionsChanged()` を経由せず状態更新が不完全

### やらないこと（スコープ外）
- 新機能追加
- UI/デザインの変更

---

## 完了条件

- [x] 比較表のチェックマークが毎回同じ表示になる
- [x] チャット中にAIが構成を生成してもチャットタブに留まれる
- [x] ショートカットパネルの Ctrl+S 説明が「プロジェクト保存」になっている
- [x] アプリ起動時に前回のプロジェクトが復元された際、CVRスコアが表示される
- [x] テンプレートマーケットプレイスからテンプレートを適用した際、CVR表示・タブ・空プレビューが正しく更新される

## 作業ログ

### Bug1: 比較表の Math.random()
- 行1733: `const isChecked = ci === 1 || Math.random() > 0.4;` → 決定論的なパターンに変更
- 原因: レンダリングのたびにチェックマークが変わる

### Bug2: onSectionsChanged の強制タブ切替
- 行1331: `switchTab('structure')` が毎回実行される
- 修正: チャットタブにいる場合はタブを切り替えない

### Bug3: ショートカットパネルの誤表記
- 行2019: `Ctrl + S → SVG ダウンロード` を `プロジェクト保存` に修正

### Bug4: 自動復元時のCVR未表示
- 行5312-5326: `updateCVRDisplay()` の呼び出しを追加

### Bug5: useMarketTemplate の不完全な状態更新
- 行4816-4828: 手動のrender呼び出しを `onSectionsChanged()` に統一
