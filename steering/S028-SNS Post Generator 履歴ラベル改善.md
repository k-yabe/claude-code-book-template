## 目的

履歴パネルに表示される入力ラベルが URL のみで分かりにくい。記事タイトルとプラットフォーム情報をラベルとして表示することで、過去の生成結果をすぐに識別できるようにする。

---

## スコープ

### やること
- [x] `HistoryManager.save()` の第1引数（input）を URL → 「プラットフォーム ｜ 記事タイトル」形式に変更
- [x] 保存するパターンを PATTERN1 固定 → おすすめ（recommend）パターンに変更
- [x] 履歴パネルの表示に platform バッジ（例: Chatter / LinkedIn / X）を追加

### やらないこと（スコープ外）
- `assets/history.js` の共通コンポーネント自体の改修（他アプリへの影響を避ける）
- 3パターン全部を履歴に保存する（UXが複雑になる）
- サーバー側への保存・同期

---

## 完了条件

- [x] 履歴パネルに「Chatter ｜ 記事タイトル（最大30文字）」のようなラベルが表示される
- [x] コピーされる内容がおすすめパターンの投稿文になっている
- [x] 既存の URL 履歴・他アプリの履歴に影響がない
