## 目的

マーケター・営業担当が、背景・対象・メッセージなどの情報を入力するだけで、AKKODiSブランドガイドライン準拠のPPTXファイルを自動生成できるツールを提供する。ウィザードによる入力補助・ブラウザプレビュー・図版自動生成・対話型リファインを備えたハイスペック構成。

## 技術仕様

### アーキテクチャ概要

```
[ウィザード入力]
      ↓
[/api/slide-generate.js (Vercel Edge Function)]
  └── ANTHROPIC_API_KEY はサーバーサイドで保持（フロントに露出させない）
  └── claude-sonnet-4-6 でスライド構成JSON生成
      ↓
[図版生成（フロントエンド）]
  ├── Chart.js → Canvas描画（display:block状態で実行）→ toDataURL() → PNG base64
  ├── Canvas直接描画（フロー図） → PNG変換
  └── 比較表 → テーブルXML文字列として生成
      ↓
[JSZip] → templates/ からテンプレートPPTX fetch → ArrayBuffer展開（キャッシュ機能あり）
  └── slide XML内プレースホルダーを書き換え
  └── [Content_Types].xml + _rels も更新してメディア追加
  └── generateAsync(DEFLATE) → Blob → ダウンロード
      ↓
[ブラウザプレビュー] ← HTML/CSSでスライドをレンダリング（クリックで詳細モーダル）
      ↓
[対話型リファイン] → /api/slide-generate.js → 修正JSON → プレビュー更新
```

### テンプレートPPTX配置方針
- 4ファイルを `apps/slide-maker/templates/` に配置してgit管理
- `fetch('templates/external-white.pptx')` でArrayBuffer取得（名前はシンプル化）
- **プレースホルダー**: テンプレートのテキスト文字列を直接置換（`Presentation Title`, `Chapter Title` など）

### Claude API 利用方針
- **キー管理**: Vercel環境変数 `ANTHROPIC_API_KEY` をVercel Edge Function内でのみ使用
- スライド構成: `claude-sonnet-4-6`（temperature=0.3）
- リファイン: `claude-haiku-4-5-20251001`（temperature=0.2、max_tokens=6000）
- システムプロンプト: 1スライド1メッセージ原則・agendaとchapterの完全一致・具体性重視

### 実装レイアウト（8種）
1. `cover` — 表紙
2. `agenda` — アジェンダ
3. `chapter` — 章区切り（ゴールドアクセント）
4. `content` — タイトル＋本文（1カラム）
5. `content-with-chart` — 本文＋データグラフ（Chart.js PNG）
6. `content-with-flow` — 本文＋フロー図（Canvas→PNG）
7. `comparison` — 比較表・マトリックス（PPTXテーブル）
8. `closing` — 締め

### 主要バグ修正（実装中に発見・修正済み）
- `Content_Types.xml` の正規表現バグ（`[^/]*` → `[^>]*`）: 旧スライドエントリが削除されない
- `slideRelsCache` が undefined のときのクラッシュ → 空Relationshipsをフォールバックに
- DEFLATE圧縮追加（ファイルサイズ削減）
- `addTitleShape` の id=901 → 9901（既存shape IDとの衝突リスク軽減）

- スライドサムネイルクリックで詳細モーダル
